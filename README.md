# Eczcalibur

Eczcalibur is a mobile app for families managing childhood eczema. It gives children an age-appropriate interface for tracking flare-ups and following treatment steps, while giving parents a clinical overview, action plan management, and a direct line to an AI assistant for appointment preparation.

---

## What it does

**Child view**
- Logs flare-ups: mood, itch intensity, affected body areas, optional photo
- Follows a personalised action plan (green / yellow / red zone steps generated by Claude)
- Earns gold coins for consistent logging, redeemable for parent-approved prizes
- Emergency screen with step-by-step red zone instructions and a one-tap parent call

**Parent view**
- Onboarding wizard: captures child profile, triggers, medications, and location; fetches live weather data; calls Claude to generate a full three-zone action plan
- Flare log history with zone filtering
- Pre-appointment summary: generates a clinical-style report for the next dermatology visit
- Claude chat: ask anything about the child's eczema history
- Prize management and redemption approval queue

---

## Who it is for

Parents of children (roughly ages 5-14) with atopic dermatitis who want a structured way to track symptoms between clinic appointments and involve the child in their own care.

---

## Why it exists

Eczema management is largely done at home, between appointments. Parents often arrive at clinic visits unable to recall whether triggers or flare patterns changed over the past three months. Children, especially younger ones, have no way to signal how they are doing between check-ins. Eczcalibur fills that gap: it records what actually happened, frames it in terms children understand, and turns that data into something a clinician can use in under ten minutes.

## Tech stack

| Layer | Choice |
|---|---|
| Framework | Expo SDK 54, React Native, expo-router v3 |
| Auth | Clerk (email/password) |
| State | Zustand with AsyncStorage persistence |

## Running the app locally

**Prerequisites**

- Node 18+
- Expo CLI: `npm install -g expo-cli`
- Expo Go installed on a physical device, or an Android emulator / iOS simulator running

**Setup**

```bash
git clone https://github.com/kelvinzigah/Eczalibur.git
cd Eczalibur
npm install
```

Create a `.env` file in the project root:

```
EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY=your_clerk_publishable_key
ANTHROPIC_API_KEY=your_anthropic_api_key
```

**Start**

```bash
npx expo start --clear
```

Press `a` for Android emulator, `i` for iOS simulator, or scan the QR code with Expo Go.

## Testing the full flow

**1. Sign up**
Create an account on the login screen. Clerk handles authentication.

**2. Complete onboarding (8 steps)**
- Enter the child's name, age, gender, and location
- Select affected body areas on the interactive body map
- Add medications (name, zone, frequency)
- Select known triggers
- Review and confirm consent
- Wait for Claude to generate the action plan (requires `ANTHROPIC_API_KEY`)
- Review the generated green / yellow / red zone plan
- Set up default prizes (or edit the names and costs)

**3. Child view**
From the parent Overview tab, tap "Switch to Child View". Enter the PIN when prompted (set on first access). The child home shows the hero canvas, heart gauge, and quest card scroller.

**4. Log a flare**
In the child view, tap "Log Flare" or use the HERO tab. Select mood, itch level, body areas, and optionally take a photo. Coins are awarded on save.

**5. Parent review**
Return to parent view (tap back or re-enter PIN). Check the LOGS tab for the new entry. Go to REPORTS, enter today's date, and generate an appointment summary.

**6. Reset for re-testing**
On the Overview tab, tap "Reset and Re-onboard". Confirm the alert. The app returns to the onboarding wizard with all data cleared.
