# Eczcalibur

Eczcalibur is a mobile app for families managing childhood eczema. It gives children (ages 7–14) a gamified quest interface for tracking flare-ups and following treatment steps, while giving parents a clinical dashboard, AI-generated action plans, and tools for appointment preparation.

Built for the **CBC McGill Hackathon 2026 — Biology & Health track**.

---

## What it does

**Child view**
- Logs flare-ups: mood/itch intensity (1–5), affected body areas, optional photo
- Follows a personalised three-zone action plan (green / yellow / red) generated by Claude
- Earns gold coins for consistent logging, redeemable for parent-approved prizes
- Emergency screen with step-by-step red zone instructions and a one-tap parent call
- Special Mission: optional watch photo logging when parent has an active skin monitor

**Parent view**
- Onboarding wizard: child profile, triggers, medications, location; fetches live weather; generates a full NEA WAP-style action plan via Claude
- Dashboard: frosted-glass card system with zone history mini-chart, ECZ Insights, Quick Actions, and WAP bullet preview
- Flare log history with zone filtering (green / yellow / red)
- **Watches (MAVL)**: monitor a specific skin area over 7/14/21 days — child logs progress photos, Claude analyses the full sequence multi-image
- Pre-appointment summary: Claude generates a clinical-style report for the next dermatology visit
- Claude chat: full conversation with context from the child's recent logs and profile
- Prize management and redemption approval queue

---

## Who it is for

Parents of children (ages 7–14) with atopic dermatitis who want a structured way to track symptoms between clinic appointments and involve the child in their own care.

---

## Why it exists

Eczema management is largely done at home, between appointments. Parents often arrive at clinic visits unable to recall whether triggers or flare patterns changed over the past three months. Children have no way to signal how they are doing between check-ins. Eczcalibur fills that gap: it records what actually happened, frames it in terms children understand, and turns that data into something a clinician can use in under ten minutes.

## Tech stack

| Layer | Choice |
|---|---|
| Framework | Expo SDK 54, React Native, Expo Router v3 |
| Auth | Clerk (email/password for parent; 4-digit PIN gate for child) |
| Database | Supabase (PostgreSQL + Realtime; RLS via Clerk JWT) |
| State | Zustand — hydrated from Supabase on launch, AsyncStorage fallback |
| AI | Anthropic Claude (`claude-opus-4-6`) via FastAPI on Fly.io |
| Weather | Open-Meteo (live humidity + temperature injected into plan generation) |
| Backend | FastAPI on Fly.io (`https://eczcalibur-api.fly.dev`) |
| Language | TypeScript — strict, 0 errors |

## Running the app locally

**Prerequisites**
- Node 18+
- Expo CLI: `npm install -g expo-cli`
- Android emulator or physical device with Expo Go

**Setup**

```bash
git clone https://github.com/kelvinzigah/Eczalibur.git
cd Eczalibur
npm install
```

Create a `.env` file in the project root:

```
EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY=your_clerk_publishable_key
CLERK_SECRET_KEY=your_clerk_secret_key
ANTHROPIC_API_KEY=your_anthropic_api_key
EXPO_PUBLIC_SUPABASE_URL=your_supabase_url
EXPO_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
EXPO_PUBLIC_API_BASE_URL=https://eczcalibur-api.fly.dev
```

**Start**

```bash
npx expo start --port 8081
# For two-device / iPhone testing:
npx expo start --tunnel
```

For Android via USB (PowerShell):
```powershell
& "<path-to-adb.exe>" reverse tcp:8081 tcp:8081
```

## Testing the full flow

**1. Sign up**
Create a parent account on the sign-in screen. Clerk handles authentication.

**2. Complete onboarding (9 steps)**
- Child's name, age, gender, location
- Affected body areas on the interactive body map
- Medications (name, frequency, instructions)
- Known triggers
- Parent name and relationship
- Consent review
- Claude generates the three-zone action plan (requires API keys)
- Review the green / yellow / red zone plan
- Set up default prizes

**3. Child view**
From the Overview tab, tap "Switch to Child View". Set a 4-digit PIN on first access. The child home shows the hero scene, heart gauge, and quest cards.

**4. Log a flare**
In the child view, use the Log tab. Select mood (1–5), body areas, optional photo. Up to 15 logs/day with a 15-second cooldown. Coins are awarded on save.

**5. Parent dashboard**
Return to parent view. The Overview tab shows zone history, ECZ Insights, and the active watch banner (if a watch is running).

**6. Watches (MAVL)**
Tap the Watches tab to view all skin monitors. Create a new watch from the dashboard or the Watches tab — pick an area and a 7/14/21-day duration. The child sees a "Special Mission" step in their log to submit a photo. Return to Watches → tap the active watch → Run Analysis to get a Claude multi-image assessment.

**7. Appointment summary**
Go to Reports, enter the appointment date, and generate a clinical summary to bring to the dermatologist.

**8. Reset for re-testing**
Settings → Reset and Re-onboard. Clears all local data and returns to onboarding.
